# Schema Changelog and Compatibility Matrix

This document records the public compatibility posture for the open schema surface. It is a review aid, not a protocol freeze. The default path remains local-first, file-first, no API key, no network calls, public fixtures only, and no trainable export artifacts.

## Compatibility Classes

| Class | Meaning | Default handling |
|---|---|---|
| Compatible additive | Adds optional information without changing old meanings. | Allowed with docs and examples. |
| Fail-closed additive | Adds a controlled value that old readers must treat as non-supporting or unknown. | Allowed only with explicit reader behavior. |
| Breaking | Renames, removes, or changes the meaning of existing required data. | Requires a new schema version and migration note. |
| Deferred | Crosses the v1 safety boundary. | Keep out of runnable manifests. |

## Compatibility Rules

Required fields:

- Adding a required field to an existing schema version is breaking.
- Removing a required field is breaking.
- Renaming a required field is breaking.

Optional fields:

- Adding an optional field is compatible when old readers can ignore it.
- Moving data from one optional field to another is breaking if old readers lose meaning.
- New nested diagnostics are preferred over new top-level fields for strict schemas.

Controlled values:

- New enum values are fail-closed unless old readers already treat unknown values as unknown.
- New support classes must not default to positive support.
- New evidence kinds need fixture coverage before they affect decisions.

Evidence and privacy:

- Source references must stay durable.
- Private source text should not be copied into portable examples.
- Redaction metadata is best-effort and must not be treated as upload approval.

Evidence Pack v1:

- New core transforms require an allowlist update, schema update, and strict audit test.
- New output schema refs require a compatibility note.
- Command packs and network packs are deferred.
- Trainable export paths are outside the v1 boundary.
